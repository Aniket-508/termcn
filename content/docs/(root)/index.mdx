**termcn** helps you run a custom [shadcn-style registry](https://ui.shadcn.com/docs/registry): you own the component source, and users install with `npx shadcn add`.

## What You Get

- **Simple layout** — Components in `registry/new-york/` plus `registry.json`
- **Documentation** — Fumadocs
- **CLI compatible** — Standard registry JSON under `public/r/`
- **Deploy ready** — Next.js app + static registry artifacts

## How It Works

1. Edit your component at `registry/new-york/your-component.tsx`
2. Update `registry.json` with names and file paths
3. Run `pnpm registry:build`

Deploy the site; share install URLs:

```bash
npx shadcn@latest add https://your-site.com/r/your-component.json
```

Why copy/paste instead of npm packages?
</AccordionTrigger>

The shadcn workflow favors **owning** the code in your app so you can change primitives, styling, and behavior without waiting on package releases.

Which frameworks are supported?
</AccordionTrigger>

Any React setup the shadcn CLI supports (Next.js, Vite, etc.), per [upstream docs](https://ui.shadcn.com/docs).

</AccordionContent>
</AccordionItem>

Yes. Add more files under `registry/new-york/` and register each item in `registry.json`.
